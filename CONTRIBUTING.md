# ü§ù Contributing to Headless Calendar

Thank you for considering contributing! We welcome all contributions, big or small.

## üöÄ Setting up the Development Environment

Our project uses **pnpm** for package management. Please ensure you have it installed globally.

1.  **Install pnpm (if necessary):**

    ```bash
    npm install -g pnpm
    ```

2.  **Fork the Repository:** Fork the repository on GitHub.

3.  **Clone Your Fork:** Clone your forked repository to your local machine.

    ```bash
    git clone https://github.com/VerbPatch/headless-calendar.git
    cd headless-calendar
    ```

4.  **Install Dependencies:** Use pnpm to install all project dependencies.

    ```bash
    pnpm install
    ```

    > **Note:** We recommend using pnpm to ensure dependency resolution is consistent and efficient.

5.  **Create a New Branch:**
    ```bash
    git checkout -b feature/my-new-feature
    # or
    git checkout -b fix/issue-number
    ```

## üêõ Reporting Bugs

If you find a bug, please help us by reporting it.

1.  **Check Existing Issues:** Before submitting, please check the [Issue Tracker](https://github.com/VerbPatch/headless-calendar/issues) to see if the issue has already been reported.
2.  **Use a Clear Title:** Give your issue a clear and descriptive title (e.g., "Error when loading data on iOS 17").
3.  **Describe the Bug:** Provide a detailed description of the problem, including:
    - Steps to reproduce the error.
    - Expected behavior.
    - Actual behavior.
    - The version of the library you are using.
    - Your operating system and environment details.

## ‚ú® Suggesting Enhancements

We appreciate feature requests and suggestions!

1.  **Check Existing Issues:** Search the [Issue Tracker](https://github.com/VerbPatch/headless-calendar/issues) to avoid duplicates.
2.  **Use a Clear Title:** Give your suggestion a clear and concise title (e.g., "Feature: Add support for X format").
3.  **Explain the Feature:** Describe the feature you'd like to see and explain why it would be useful to other users.

## üíª Code Contributions (Pull Requests)

We follow a standard Git workflow for contributions.

1.  **Make Changes:** Write your code, tests, and documentation.
2.  **Commit Your Changes:** Write a clear, descriptive commit message. We recommend using [Conventional Commits](https://www.conventionalcommits.org/en/v1.0.0/).
    - **Example:** `feat: add support for new configuration option`
3.  **Push to Your Fork:**
    ```bash
    git push origin feature/my-new-feature
    ```
4.  **Submit a Pull Request (PR):** Open a PR against the `main` branch of the original repository.
